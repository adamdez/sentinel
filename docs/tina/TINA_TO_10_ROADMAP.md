## Scale

- `8` = extraordinary 100-year CPA veteran ceiling
- `10` = that ceiling plus AI leverage

Rule:

- passing a gate is not enough
- shipping a feature is not enough
- Tina only earns higher scores when engine-level capability and reviewer-trust outcomes both improve

## What keeps Tina below 8 today

These are the systems that must exist before Tina can honestly clear the human-veteran bar across the weakest current skills.

### P0: Reach 8+

1. Reviewer outcome memory and learning loop
Reason:
Tina cannot claim elite judgment if reviewer changes vanish after the file closes.

Must build:
- persistent reviewer decision records tied to issue, treatment, and output
- explicit override objects with before and after state
- reusable outcome-pattern summaries for future files

Skills unlocked:
- reviewer learning loop
- reviewer-override governance
- live acceptance testing against reality
- durable case memory
- practice judgment

2. Transaction-level tie-out engine
Reason:
Accounting fluency will stay capped until every material amount is provable from raw records.

Must build:
- normalized transaction or ledger ingestion
- deterministic tie-out from source records to workpaper lines to return-facing amounts
- explicit mismatch classes for duplicates, scale errors, contamination, and unsupported balances

Skills unlocked:
- accounting fluency
- record and evidence analysis
- form and compliance execution
- documentation and defensibility
- messy-evidence generalization

3. Cross-entity normalization and scope control
Reason:
Commingled books and owner-flow contamination are still one of the hardest ceilings on Tina's real-world trust.

Must build:
- entity mapping across documents, books, EINs, and return scope
- intercompany and owner-flow separation logs
- mechanical inclusion and exclusion reasoning for each record

Skills unlocked:
- accounting fluency
- fact-pattern judgment
- entity and filing-path classification
- risk and materiality judgment
- unknown-pattern resolution

Exit condition for P0:
- Tina can learn from reviewer changes
- Tina can prove material numbers
- Tina can resolve cross-entity contamination instead of only detecting it

## What keeps Tina below 9 after that

These are the systems that separate an extraordinary guarded engine from a broadly elite practitioner-grade one.

### P1: Reach 9

1. Multi-lane federal return completion
Must build:
- deterministic `1120-S` draft and readiness path
- deterministic `1065` draft and readiness path
- lane-specific package rules and reviewer packet outputs

2. Basis, capital, debt basis, and owner-account modeling
Must build:
- shareholder basis rollforwards
- partner capital and debt-basis rollforwards
- owner draw, contribution, distribution, and loan characterization engine

3. Authority-grade position memory
Must build:
- versioned position records tied to facts, authority level, confidence, and disclosure consequence
- reviewer decision deltas on each position
- return-impact rules that only activate approved supported positions

4. Deep document-intelligence expansion
Must build:
- richer parsing for prior returns, elections, ownership records, payroll, and fixed-asset support
- completeness scoring for partial or low-trust document sets
- cross-document contradiction graphs instead of isolated clue rows

Exit condition for P1:
- Tina can handle broad federal-family execution
- Tina can resolve owner-account and basis-heavy files numerically
- Tina can defend positions with durable legal memory

## What keeps Tina below 10+

This is where “excellent software” stops and “better than an extraordinary human veteran” begins.

### P2: Reach 10+

1. Live acceptance benchmarking against real reviewer outcomes
Must build:
- acceptance ledger for accepted, revised, and rejected outputs
- skill-by-skill rescoring rules tied to real reviewer windows
- benchmark dashboards showing where reviewers still disagree with Tina

2. Planning depth and advisor-grade tradeoff reasoning
Must build:
- structured planning scenarios with facts, authority posture, and payoff windows
- tradeoff comparisons between legal paths
- sequenced advice that balances correctness, usefulness, and business practicality

Exit condition for P2:
- Tina wins not only on control and discipline but on repeated reviewer-trusted outcomes
- `10+` claims are backed by real acceptance data, not by internal confidence

## Re-score rules

Use these rules whenever another agent wants to move a benchmark score:

1. Do not move a score because a screen exists.
2. Do not move a score because a route passes tests.
3. Move a score only when the underlying engine changed in a way that broadens real file capability.
4. For any move above `8`, require evidence that Tina is outperforming a human-veteran baseline for that skill.
5. For any move to `10+`, require real reviewer-trust outcome evidence, not just internal correctness.

## Current build order recommendation

If the goal is to deliver Tina as fast as possible without lying about quality, the order should be:

1. Reviewer outcome memory and override governance
2. Transaction-level tie-out
3. Cross-entity normalization
4. Multi-lane federal completion
5. Basis and owner-account modeling
6. Authority-grade position memory
7. Deep document-intelligence expansion
8. Live acceptance benchmarking
9. Advisor-grade planning depth

## Bottom line

Tina does not need a thousand more features to reach the benchmark ceiling.

She needs a smaller number of heavy systems that make her:

- numerically provable
- reviewer-traceable
- cross-entity safe
- multi-lane complete
- outcome-measured against reality

That is the path from "strong guarded prep engine" to honest `10+` delivery.

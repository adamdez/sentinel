# Tina +2 To +4 Movement Plan

Date: 2026-04-07

This plan is for honest score movement only. A category does not move because a feature exists. It moves when the macro-pass evidence changes real behavior, survives adversarial tests, and improves reviewer trust.

## Scoring Rule

- `+1`: the engine exists and changes behavior in a narrow supported lane
- `+2`: the engine is real, tested, and reliably changes outcomes in messy but still bounded cases
- `+3`: the engine is durable across messy/adversarial files and meaningfully reduces reviewer corrections
- `+4`: reserved for categories where the macro pass creates a true step-change in reviewer trust or packet quality across the whole supported lane

## Macro Passes

- `Macro Pass A`: Transaction Truth Engine
- `Macro Pass B`: Reviewer Reality Loop
- `Macro Pass C`: CPA-Ready Final Package Engine
